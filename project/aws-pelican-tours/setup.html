🎓 Your AWS Academy Learner Lab

Real AWS console, real services, $100 of class credit, zero credit cards. ~20 minutes to get oriented — do this before Class 4 — M1 is due then and requires it. The lab has house rules — read them first.

Getting in

  • Accept the AWS Academy email invitation from your instructor and create your Academy (Canvas) account.
  • Open the course → Modules → Launch AWS Academy Learner Lab → read/accept the terms.
  • On the lab page, press ▶ Start Lab. Wait for the circle next to "AWS" to turn green, then click the green AWS link — the real AWS console opens in a new tab, already signed in.
  • You're in as a federated, temporary identity using a role called LabRole. Note it — it's an M2 deliverable.
✅ Done when: the console is open, region shows us-east-1 (N. Virginia), and you can see your remaining budget on the lab page. Screenshot both for your journal — that budget number is your first "measured service" reading.

House rules of the sandbox

RuleWhat it means for you
$100 budget, hard stopShown on the lab page (updates every ~8–12 hrs, not live). If you hit it, the account is disabled and your work is gone — there is no appeal desk. The project is designed to use a fraction of it.
Sessions endThe lab session timer (usually 4 hrs) stops your session: EC2 instances are stopped (not terminated). Press Start Lab next time and they come back — with new public IPs. Use the ALB DNS name, not instance IPs, for anything you save or bookmark.
Regionsus-east-1 only (plus us-west-2 where M3 says so). Resources elsewhere may be blocked — and anything orphaned there still burns budget.
IAM is read-mostlyYou can't create users, and role creation is restricted. When anything asks for a role, use LabRole; for EC2 instance profiles use LabInstanceProfile. This is least privilege applied to lab accounts.
Some services are off-limitsBilling console, Budgets, Route 53, and various others are blocked or limited; a few (like Cognito) vary by lab template — where a milestone touches one of these, the brief includes a fallback. "Access Denied" on a milestone service? Journal it and use the fallback.
Instance limitsOnly small instance types are allowed (and the project only ever needs t3.micro/t3.small). The lab caps how many can run at once — another reason desired capacity goes back to 1.
🔒 Cost & safety rules: t3.micro / t3.small only — nothing bigger; no NAT gateways, no unused Elastic IPs, no GPU anything. Security groups: 80/443 open to the world at most; SSH (22) only from your IP. Fictional guest data only — never real personal information in the lab.

The end-of-session ritual

Before you close the laptop, every single time:
 1. Auto Scaling group desired capacity → 1 (or 0 if done for the day)
 2. Terminate anything you experimented with and no longer need
 3. Check us-west-2 is empty (unless mid-M3)
 4. Lab page → End Lab   (instances stop; the meter mostly stops with them)
 5. Journal entry: what ran, what you stopped, budget reading

S3, DynamoDB, EBS volumes, and snapshots keep existing (and gently billing) while the lab is off — that's fine, they're pennies. The costly items are instances, load balancers, EFS mount targets, and forgotten experiments. The ALB is the priciest always-on piece (~$16/month if left up 24/7 — but your lab only "runs" while started, so in practice it costs a fraction of that; still, delete it if you'll be away a week).

SSH keys & connecting to instances

  • The lab provides a ready key pair named vockey — download the .pem (Mac/Linux) or .ppk (PuTTY) from the lab page ("AWS Details" → SSH key).
  • Easier: skip SSH entirely with EC2 Instance Connect (Connect button in the console) or Session Manager — no keys, no port 22 open to the world — note the security difference in your journal.
chmod 400 ~/Downloads/labsuser.pem
ssh -i ~/Downloads/labsuser.pem ec2-user@INSTANCE-PUBLIC-IP
# app logs:      sudo journalctl -u pelican -f
# app restart:   sudo systemctl restart pelican

Troubleshooting the greatest hits

SymptomLikely cause → fix
Green AWS dot never appearsPop-up blocker ate the console tab, or the lab is still provisioning — wait 2 min, allow pop-ups for the Academy site, press Start Lab again.
Site worked yesterday, dead todaySession ended → instance stopped → new public IP on restart. Get the new IP, or (better) use the ALB DNS name which never changes.
"Access Denied" in the consoleYou've hit a Learner Lab restriction. Check the region first (us-east-1?), then the service list on the lab's README. If a milestone service is blocked, use the brief's fallback and journal it.
Role dropdown is empty / role errorUse LabRole (services) or LabInstanceProfile (EC2). You cannot create new roles — that's by design.
User data "didn't run"It runs once, at first boot only. Check /var/log/cloud-init-output.log; did you replace APP_ZIP_URL? Relaunch the instance rather than fighting it.
Site loads, tours don't (S3 frontend)Browser console will show it: config.js missing/wrong API_BASE, or CORS — confirm the API sends the CORS headers (starter app does) and API Gateway has CORS enabled (M4).
S3 website gives 403Static website hosting on, bucket policy allowing public read on that bucket, and that bucket's Block Public Access turned off (only that one — M3 audits this).
Budget dropped more than expectedSomething ran while you were gone. EC2 → all regions check (us-west-2!), delete orphans, and journal the post-mortem.

Golden rule: read the exact error text and check the region first — most lab issues are a wrong region or a stopped session. When searching for help, include the exact error message and what you clicked.

No Learner Lab? PERSONAL ACCOUNT APPENDIX

If you're doing this outside a class Learner Lab: AWS's free plan for new accounts gives $100 in credits (up to $200 with activity bonuses) for 6 months, and on the free account plan you can't be charged — the account simply closes when credits or months run out. Signup does require a credit card and a phone number.

  • Everything in this project fits comfortably in those credits — the budget table above still applies.
  • Differences from the Learner Lab: you have full IAM (create a non-root admin user with MFA on day one — that becomes your M2 evidence), all regions (stay in us-east-1 anyway), and Budgets works — set a $10 alert immediately.
  • The end-of-session ritual matters more, not less. Set a calendar reminder to close/clean the account at semester's end.
