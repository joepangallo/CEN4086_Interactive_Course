☁️ Pocket Cloud — You Are the Provider Now

Every session in this course you study clouds. In this project, you run one. Over four weeks your laptop becomes a working miniature cloud provider — with real tenants, real identity, real storage, real outages, and real invoices.

Individual project4 milestones · Classes 4 · 7 · 9 · 11Runs 100% on your own machineFree tools only — no credit card, everLive demo on Class 12

The story

You've founded a one-person hosting company: Pocket Cloud. Your data center is your laptop.

Two local businesses just signed as your first tenants:

  • Palmetto Surf Co. — a Cocoa Beach surf shop that needs an online storefront. Traffic spikes every time the surf report is good.
  • Mango Yoga Studio — a tiny studio that wants the same storefront product. They will not pay for a second codebase.

Over four weeks you'll take Pocket Cloud from "one container" to a platform with self-service provisioning, load balancing, single sign-on with MFA, object storage, backups that survive a disaster you inflict on yourself, monitoring with alerts, a REST API — and at the end, you'll bill your tenants from real usage data and present the whole thing live.

At every step, connect what you build to the concept it implements — "this is rapid elasticity, and here is my evidence."

# Caddyfile — SSO in front of both tenants (Authelia 4.38+)
# auth.localhost, palmetto.localhost, mango.localhost all resolve to
# 127.0.0.1 automatically — and Caddy gives them local HTTPS for free.
auth.localhost {
  reverse_proxy authelia:9091
}
palmetto.localhost {
  forward_auth authelia:9091 {
    uri /api/authz/forward-auth
    copy_headers Remote-User Remote-Groups Remote-Name Remote-Email
  }
  reverse_proxy palmetto:3000
}
mango.localhost {
  forward_auth authelia:9091 {
    uri /api/authz/forward-auth
    copy_headers Remote-User Remote-Groups Remote-Name Remote-Email
  }
  reverse_proxy mango:3000
}
