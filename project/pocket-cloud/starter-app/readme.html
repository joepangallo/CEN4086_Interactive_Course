📦 The tenant storefront

The app you'll host on your Pocket Cloud. Zero dependencies — Node's standard library only. Files: server.js · products.json · products-yoga.json · Dockerfile · public/index.html

Run it bare first (before Docker, just to see it work)

node server.js
# → http://localhost:3000

Build & run as a container

docker build -t storefront:1.0 .
docker run -d --name palmetto -p 8081:3000 storefront:1.0
# → http://localhost:8081

Second tenant, same image (multi-tenancy)

docker run -d --name mango -p 8082:3000 \
  -e TENANT_NAME="Mango Yoga Studio" \
  -e THEME_COLOR="#e0813a" \
  -e TAGLINE="Breathe in, stream on" \
  -v "$(pwd)/products-yoga.json:/app/products.json:ro" \
  storefront:1.0
# → http://localhost:8082  — same code, different tenant

One image, two customers: branding via environment variables, data via a mounted file. That pattern is the M1 multi-tenancy discussion.
