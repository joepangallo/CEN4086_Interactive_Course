🛠 Setting up your local server

One-time setup, ~30–45 minutes. Do this before Class 4 — M1 is due then and requires it. Your laptop becomes the data center — no accounts with a credit card, no paid anything, ever.

What your machine needs

MinimumComfortable
RAM8 GB16 GB
Free disk15 GB25 GB
OSWindows 10/11 (64-bit) · macOS 12+ (Intel or Apple Silicon) · any mainstream Linux
CPUAnything from the last ~8 years with virtualization support (nearly all have it)

Under 8 GB RAM, or a school-locked machine where you can't install software? Skip to Plan B / Plan C below — you can complete the entire project in a free browser-based environment.

Plan A · Docker Desktop on your machine RECOMMENDED

Windows

  • Download Docker Desktop for Windows from docker.com/products/docker-desktop (free for personal & education use — no account needed to download or run).
  • The installer enables WSL 2 (Windows Subsystem for Linux). Accept it; reboot when asked.
  • If it complains about virtualization: enable Intel VT-x / AMD-V (SVM) in your BIOS/UEFI — usually under "Advanced" or "CPU Configuration."

macOS

  • Download Docker Desktop for Mac — pick Apple Silicon or Intel to match your machine ( → About This Mac).
  • Drag to Applications, open, grant the permissions it asks for.

Linux

  • Install Docker Engine + the compose plugin from your distro's docs (docs.docker.com/engine/install), then add yourself to the docker group: sudo usermod -aG docker $USER and log out/in.
💡 Prefer Podman (or Docker Desktop won't run for you)? Podman Desktop is a drop-in for this project — podman compose replaces docker compose. Note any differences you hit in your journal.

Verify it works (this screenshot is checkpoint 1)

docker --version
docker compose version
docker run hello-world
docker run -d -p 8088:80 --name web-test nginx:alpine
# open http://localhost:8088 and confirm the nginx welcome page appears
docker rm -f web-test

Install your cloud console (Portainer CE)

docker volume create portainer_data
docker run -d -p 9443:9443 --name portainer --restart=always \
  -v /var/run/docker.sock:/var/run/docker.sock \
  -v portainer_data:/data portainer/portainer-ce:latest
# open https://localhost:9443 — accept the local-certificate warning,
# create your admin account IMMEDIATELY (it locks after a few minutes),
# use a strong password you don't use anywhere else.
✅ Done when: hello-world ran, you saw nginx on port 8088, and Portainer shows your containers at https://localhost:9443. Screenshot Portainer for your journal — that's your cloud console.

Plan B · GitHub Codespaces FREE FALLBACK

  • Free personal GitHub accounts include ~60 hours/month of a 2-core, 8 GB cloud dev machine — Docker preinstalled, no credit card required.
  • Create your project repo on GitHub → green Code button → Codespaces → Create codespace on main. You get VS Code in the browser with a terminal; every docker … command in this project works there. Forwarded ports replace localhost links.
  • Budget your hours: stop the codespace when you're not working (it also auto-stops). Track usage at github.com/settings/billing — it will never charge you; it just stops at the limit.

Plan C · Play with Docker LAST RESORT

  • labs.play-with-docker.com — free browser terminal with Docker, using a free Docker Hub account.
  • Sessions last 4 hours and then everything is deleted, so keep every config in your Git repo and re-run from scratch each session.
  • Use it for in-class checkpoints in a pinch; use Plan B for milestone work.

Troubleshooting the greatest hits

SymptomLikely cause → fix
"WSL 2 installation is incomplete" (Windows)Run wsl --update in PowerShell (as admin), reboot, relaunch Docker Desktop.
"Virtualization not enabled"Enable VT-x / AMD-V in BIOS/UEFI. On school machines this may be locked → Plan B.
port is already allocatedSomething else owns that port. Pick another (-p 8089:80) or find the squatter: docker ps, or netstat -ano | findstr :8088 (Win) / lsof -i :8088 (Mac/Linux).
permission denied … docker.sock (Linux)You skipped the docker-group step, or didn't log out/in after it.
Docker Desktop eats all RAMSettings → Resources → cap memory (e.g., 4 GB). On Windows, create %UserProfile%\.wslconfig with [wsl2] / memory=4GB, then wsl --shutdown.
*.localhost names don't resolve (M2)Modern browsers/OSes send them to 127.0.0.1 automatically. If yours doesn't, add lines to your hosts file: 127.0.0.1 auth.localhost palmetto.localhost mango.localhost.
Phone can't reach the site (M1)Same Wi-Fi? Use the laptop's LAN IP (ipconfig/ifconfig), not "localhost". Campus/guest Wi-Fi often blocks device-to-device — demo on a home network or phone hotspot instead.
Nothing works anymoreRun docker compose down, then Docker Desktop → Troubleshoot → Restart. If problems persist, Reset to factory defaults erases all containers/volumes — fine early on, a disaster after M2 unless your backups work.

Golden rule: read the exact error text carefully — it usually names the fix. When searching for help, include the exact message, the command that caused it, and your OS.

Safety rules

🔒 Local means local. Never forward router ports to your Pocket Cloud or expose it to the internet (no ngrok/tunnels for this project). Strong unique passwords on every admin console (Portainer, MinIO, Authelia), stored in your password manager — never in Git. Only break, stop, or load-test containers that are yours. No tool in this project requires a credit card.
