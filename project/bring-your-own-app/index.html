📱 Bring Your Own App — Your App, Real Cloud

Already built a mobile app, or a website with an app behind it? In this track you don't get a fictional client — your own app is the client. Over four weeks you take it from "runs on my machine" to a cloud deployment with identity, storage, backups, monitoring, and a documented API.

How this track works

This is an alternative to the Pocket Cloud and Pelican Tours tracks with the same objectives, the same milestone schedule, and the same grading weight — the difference is the raw material. You bring an app you built (a mobile app, a web app, or a website with an app component), get it approved before Class 4 (email your proposal right after Class 3), and every milestone is performed on your system.

Step 1 — check eligibility and submit your proposal: the Eligibility & Setup page defines what qualifies, what you must never do with real user data, and the one-page proposal emailed right after Class 3. If your app isn't approved, you simply take one of the other two tracks — no penalty.

Step 2 — choose your platform (also in the proposal):

PlatformWhat it meansEnvironment guide
LOCAL Docker on your machineYour backend runs as containers behind a reverse proxy; MinIO for object storage; Authelia for SSO; Uptime Kuma for monitoringPocket Cloud setup guide
AWS Academy Learner LabYour backend runs on EC2 (later Lambda if you refactor); S3 for storage; CloudWatch + SNS for monitoring; $100 class credit, no credit cardLearner Lab setup guide

Mobile apps: the phone client stays on the phone or emulator — it's your server side that moves to the cloud. If your app has no server side yet, building a minimal API backend is part of M1 (the sibling tracks' starter apps show how small that can be). Milestone items below are written platform-neutral, with LOCAL/AWS markers where the tools differ.
